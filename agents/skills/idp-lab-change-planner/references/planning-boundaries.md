# Planning Boundaries

## MVP In Scope

- Port service catalog model and demo entities.
- IKS metadata visibility and scorecards.
- Low-risk workflows that create tasks, checklists, or review requests.
- Agent and workflow blueprints as future extension points.
- Documentation and ADRs that clarify operating model decisions.

## MVP Out Of Scope

- Kubernetes or runtime-health dashboards.
- Automatic infrastructure provisioning.
- Autonomous compliance enforcement.
- Direct mutation of production systems.

## ADR Triggers

- Changing the source of truth.
- Expanding or shrinking MVP scope.
- Introducing a new external integration category.
- Changing governance or approval boundaries.
