# IKS Metadata Review Checklist

- Use `agents/checklists/code-review-pyramid.md`: report metadata contract and semantic readiness issues before style or wording suggestions.
- Confirm the service has technical owner, business owner, lifecycle, repository, documentation, criticality, protection need, and data class.
- Treat runbook as expected operational hygiene; report when empty even if not blueprint-required.
- Check `complianceScope` for `iks` when the service is IKS-relevant.
- Verify relations point to existing team, system, and repository entities.
- Prefer concrete patch suggestions over broad governance commentary.
