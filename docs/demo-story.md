# Demo-Story

## Persona

Eine Plattform- oder IKS-verantwortliche Person moechte schnell erkennen, welche Services existieren, wer verantwortlich ist und ob die Mindestmetadaten fuer IKS gepflegt sind.

## Ablauf

1. Port oeffnet den Service-Katalog.
2. Die Ansicht wird nach kritischen oder IKS-relevanten Services gefiltert.
3. Der Service `customer-portal` wird geoeffnet.
4. Owner, fachlicher Owner, Repository und Dokumentation sind sichtbar.
5. Die IKS-Basis-Scorecard zeigt, ob Schutzbedarf, Datenklasse und Kritikalitaet gepflegt sind.
6. Beim Service `reporting-api` fehlt bewusst ein Runbook-Link.
7. Eine Action erzeugt ein GitHub Issue zur Nachpflege.
8. Die Agenten-Blueprints zeigen, welche Agenten spaeter mit diesem Kontext arbeiten duerfen.

## Erfolgsbild

Die Demo zeigt keinen fertigen Plattformbetrieb, sondern einen klaren, anschlussfaehigen Start: Katalogdaten, Standards, einfache Workflows und ein Weg zu spaeteren Agenten.
