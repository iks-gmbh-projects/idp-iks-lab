# Operating Model

## Rollen

- Platform Owner: priorisiert Portal- und Workflow-Faehigkeiten.
- Catalog Maintainer: pflegt Blueprints, Entities und Datenqualitaetsregeln.
- Service Owner: verantwortet Service-Metadaten, Doku und Runbooks.
- IKS Reviewer: prueft IKS-relevante Angaben und Review-Anfragen.
- Agent Owner: verantwortet spaetere Agenten, erlaubte Actions und Auditierbarkeit.

## Arbeitsweise

- Aenderungen an Katalogdaten erfolgen versioniert ueber GitHub.
- Port liest Metadaten aus GitHub oder wird aus GitHub-Konfiguration befuellt.
- Scorecards machen Luecken sichtbar, blockieren im MVP aber noch keine Teams.
- Workflows erzeugen nachvollziehbare Aufgaben statt direkt produktive Systeme zu veraendern.

## Governance im MVP

- Keine automatische Ablehnung von Services.
- Fehlende Felder werden sichtbar gemacht und per Issue nachverfolgt.
- IKS-Reviews werden als Workflow vorbereitet und koennen manuell entschieden werden.
